Heritage Vault 2.0 — Secure Linux Infrastructure Project

Heritage Vault 2.0 is one of the most meaningful and challenging projects I’ve worked on. The goal was to design a secure, scalable Linux infrastructure capable of storing sacred genealogical data. I built out automation, centralized authentication, RAID + LVM storage, SELinux/AppArmor security, VLAN segmentation, VPN access, and monitoring with Prometheus and Grafana.

Personal Story: The Night This Project Tested Me

There was a night during this project that I still think about. I had just finished a long shift, and by the time I sat down to work on the assignment, I was exhausted. A configuration kept breaking, and no matter what I tried, the system refused to cooperate.

I sat there staring at the screen, frustrated and tempted to give up for the night. But then I remembered why I started this journey — the sacrifices I made to move countries, the challenges I’ve overcome, and the determination that brought me this far.

So I took a deep breath, opened my notes, and tried again. And again. And again. Eventually, the system responded. The configuration worked. In that moment, I realised that my resilience is one of my greatest technical skills.

This project didn’t just teach me about Linux security — it taught me about myself. It reminded me that I don’t quit when things get hard. I adapt. I persist. I grow.

Ethical Logging & Privacy‑First Design

This project focused on designing a logging system that protects user privacy while still providing meaningful operational insights. I implemented minimum‑necessary data principles, removed personal identifiers, and added role‑based access controls.

What I Learned

This project helped me understand that ethical design isn’t an optional add‑on — it’s a responsibility. Technology should protect people, not expose them.

Network Segmentation & Zero‑Trust Architecture

In this project, I designed a segmented network using VLANs, firewalls, and zero‑trust principles. The goal was to reduce attack surfaces and ensure that every request is authenticated and authorized.

What I Learned

Security is not a single tool — it’s a mindset. It requires thinking ahead, anticipating risks, and designing systems that protect users even when things go wrong.

Automation with Bash & Ansible

This project involved automating system configuration and deployment using Bash scripting and Ansible playbooks. I automated user creation, package installation, service configuration, and security hardening.

What I Learned

Automation is about consistency, reliability, and reducing human error. It also taught me patience — debugging automation requires a different kind of persistence.

Monitoring & Observability with Prometheus and Grafana

I built a monitoring stack that tracks system performance, logs, and alerts. This included dashboards, alert rules, and service health checks.

What I Learned

Monitoring is the heartbeat of any infrastructure. It’s how you see problems before users feel them.
